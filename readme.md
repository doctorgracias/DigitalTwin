# Robot Navigator: Процедурная генерация сред и Any-Angle планирование пути

Программный комплекс на базе Blender API (bpy) и Python, предназначенный для автоматического создания трехмерных симуляционных пространств и расчета оптимальных траекторий движения мобильных роботов с использованием алгоритма Theta*. 

Проект разработан с учетом архитектурных стандартов экосистемы ROS 2 (Nav2) и симулятора Gazebo.

## Функционал проекта по лабораторным работам

### Лабораторная работа №1: Процедурный Asset-Pipeline и HMI
* **Графический интерфейс:** Разработана удобная N-панель управления симуляцией, встроенная в 3D Viewport Blender.
* **Автоматизация геометрии:** Реализован конвейер полной очистки контекста сцены и процедурной генерации комнат. К каждому объекту применяется сброс матриц (transform_apply), что критически важно для физических движков (ODE/Bullet) в ROS/Gazebo.
* **Семантическое именование:** Все сущности (Robot_Sim, Goal_Marker) именуются строго по регламенту, подготавливая дерево трансформаций систем координат (TF2 tree).

### Лабораторная работа №2: Дискретизация и Any-Angle навигация
* **Grid Mapping:** Алгоритм аппроксимирует 3D-меши в дискретную 2D-карту проходимости (полный аналог ROS-сообщения nav_msgs/OccupancyGrid).
* **Inflation Layer:** Параметр robot_radius выполняет роль слоя инфляции, динамически «раздувая» стены на габариты робота для обеспечения безопасности движения.
* **Планировщик Theta\*:** В отличие от классического зигзагообразного A*, алгоритм использует проверку прямой видимости (Line of Sight), за счет чего робот двигается по сглаженным траекториям под произвольными углами, сокращая путь на 11-14%.

## Тестовые сценарии (Сцены)

Для комплексного стресс-тестирования навигационного алгоритма в аддон заложены 3 конфигурации помещений:
1. **Офис:** Высокая плотность мелких препятствий, узкие коридоры и проверка прохождения робота в минимальные зазоры.
2. **Склад:** Длинные прямые логистические ряды для проверки эффективности алгоритма сглаживания пути на больших дистанциях.
3. **Лаборатория (12х12):** Квадратное помещение с изолированными центральными островками-стендами для отработки сложных маневров огибания.

## Как запустить аддон в Blender

1. Откройте Blender (версии 4.0 и выше).
2. Перейдите во вкладку Scripting и создайте новый текстовый файл.
3. Вставьте содержимое основного скрипта robot_navigator.py.
4. Нажмите кнопку Run Script (Запустить скрипт).
5. Откройте боковую панель в 3D-пространстве (клавиша N) и перейдите на вкладку Robot Nav.
6. Выберите нужную сцену, задайте параметры робота и нажмите «Создать сцену», а затем «Запустить симуляцию».
